# DataBuilder remote training bundle

This directory is a self-contained training package generated by
DataBuilder. The dataset has already been encoded on the source
machine; **the raw images and plaintext captions never left**. Only
their VAE latents and text-encoder embeddings are inside `cache/`.

## Cloud-side run (vast.ai / RunPod / Lambda / your own GPU box)

```bash
# 1. Upload this whole folder to the GPU machine (rsync / scp / web UI).
# 2. SSH in, then:
cd remote_bundle/
bash setup.sh           # creates .venv, installs deps (~2-5 min first time)
python train.py         # runs the training to completion
```

That's it. Training writes checkpoints + samples + logs to `./output/`.

## Pulling results back

```bash
# Replace <host> with your vast.ai/RunPod SSH endpoint:
rsync -avz <host>:remote_bundle/output/ ./output/
```

The trained LoRA / fine-tune weights live under `output/` alongside
the training logs.

## Privacy properties

- Only `cache/cache_*.safetensors` (latents + text embeddings) and
  `cache/captions.json` (post-processed caption *strings*) leave your
  machine. Captions are sent in plaintext because the trainer needs
  to apply tag shuffling / dropout at training time.
- If your captions contain PII you want kept private, redact them
  before building the bundle. The encoded latents are not directly
  recoverable into images without your VAE weights, but model-inversion
  attacks against VAEs do exist — treat this as "much better than
  uploading raw images" rather than "perfect privacy".
- The training_config.json snapshot includes your dataset paths only
  in the form they were originally configured (mostly local paths
  with usernames). Inspect `config/training_config.json` and edit it
  before uploading if those paths are sensitive.

## Environment knobs (override at `train.py` invocation)

| Variable             | Default | Purpose                                    |
|----------------------|---------|--------------------------------------------|
| `HF_TOKEN`           | (none)  | Required for gated repos when no `model/`  |
| `DATABUILDER_LOG`    | `INFO`  | Set `DEBUG` for verbose logging            |
| `DATABUILDER_DEVICE` | auto    | Force `cuda:0` etc. (rarely needed)        |

## Troubleshooting

- **`HF 401 / 403 on model download`** — gated repo + missing
  `HF_TOKEN`. Get one at https://huggingface.co/settings/tokens and
  re-run `bash setup.sh`.
- **`No GPU detected`** — `nvidia-smi` fails. Confirm the rented
  instance actually has a GPU attached; CPU-only training is too slow
  for this workflow.
- **`OOM after a few steps`** — your local config used a smaller
  batch size or different precision than the cloud GPU can handle.
  Edit `config/training_config.json` (`batch_size`, `precision`,
  `gradient_accumulation`) and re-run.
- **`SafetensorsMMapDataset … no cache shards found`** — the bundle
  was uploaded incomplete; re-upload `cache/` and verify
  `cache_*.safetensors` files made it across.
